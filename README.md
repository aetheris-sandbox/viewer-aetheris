# Aetheris Sandbox — 3D Viewer

Это простой статический сайт для просмотра 3D-моделей, загруженных на ваш сервер.

Что входит
- `index.html` — интерфейс (подключает `style.css` из рабочей папки).
- `viewer.js` — ES module на Three.js. Загружает список моделей, отображает их и рендерит выбранную модель.
- `models/models.json.example` — пример манифеста (см. ниже).

Как это работает
- По умолчанию сайт пытается получить список моделей по URL `models/models.json`.
- Если `models/models.json` не найден, пытается запросить `models/` и парсит HTML (подходит, если ваш сервер отдаёт индекс каталога).
- Для загрузки моделей поддерживаются `.glb`, `.gltf`, `.obj` (рекомендуется `.glb`).

Развёртывание
1. Поместите файлы сайта (включая `style.css`, `index.html`, `viewer.js`) на ваш сервер.
2. Создайте папку `models/` рядом с этими файлами и загрузите туда ваши модельные файлы.
3. (Рекомен.) Создайте `models/models.json` с перечислением файлов или объектов. Пример ниже.

Пример `models/models.json`

[
  { "name": "MyModel1", "url": "models/mymodel1.glb" },
  "another_model.glb",
  { "name": "OldObj", "url": "models/old_model.obj" }
]

Пример: если вы используете только строки, они будут интерпретироваться как относительный путь внутри `models/`.

Запуск локально (для тестирования)
- Для корректной работы ES modules и загрузки моделей лучше запускать через локальный HTTP-сервер.
- Примеры (в PowerShell):

```powershell
# Python 3
python -m http.server 8000

# Или если у вас Node.js
npx http-server -p 8000
```

После этого откройте в браузере `http://localhost:8000`.

Советы и ограничения
- Загрузчики подтягиваются через CDN (unpkg) как ES modules — требует современного браузера и работы по http(s).
- Если сервер не отдаёт индекс папки и вы не хотите создавать `models.json`, клиент не сможет получить список автоматически.
- Поддержка текстур и зависимостей для `.obj` (напр., `.mtl`) может потребовать дополнительной настройки путей.
